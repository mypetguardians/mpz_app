## 4. 인증 흐름

### 4.1 카카오 로그인
```
[사용자] → 카카오 로그인 버튼 클릭
  ├─ Web: Kakao JS SDK → OAuth 팝업 → 콜백
  └─ Native: Capacitor Plugin → KakaoNativeLogin → 토큰 반환
    → /oauth/kakao/callback
    → 백엔드 /v1/kakao/login/callback 호출
    → JWT 토큰 발급
    → access_token: 쿠키 (7일)
    → refresh_token: localStorage
```

### 4.2 센터 관리자 로그인
```
[관리자] → username/password 입력
  → 백엔드 /v1/auth/login 호출
  → JWT 토큰 발급 (동일 방식)
```

### 4.3 토큰 관리
- `AuthProvider`에서 마운트 시 `/auth/me` 호출하여 사용자 정보 로드
- access_token은 httpOnly 쿠키 (7일), refresh_token은 localStorage
- iOS Safari 호환을 위한 `storage-utils.ts` 사용 (쿠키 폴백)

## 5. 상태 관리

### 5.1 서버 상태 (TanStack React Query)
- **Query 훅**: `src/hooks/query/` (44개)
  - `useGetAnimals()` - 무한 스크롤 동물 목록
  - `useGetAnimalById()` - 동물 상세
  - `useGetMyProfile()` - 현재 사용자
  - `useGetUserAdoptions()` - 입양 내역
  - `useGetMixedPosts()` - 커뮤니티 게시글
  - 기본 설정: staleTime 5~10분, gcTime 30분, retry 1회

- **Mutation 훅**: `src/hooks/mutation/` (50개+)
  - `useCreateAnimal()`, `useUpdateAnimal()` - 동물 CRUD
  - `useAdoptionApplication()` - 입양 신청
  - `useCreatePost()`, `useCreateComment()` - 커뮤니티
  - `useToggleAnimalFavorite()` - 즐겨찾기 토글
  - `usePushToken()` - FCM 푸시 토큰 등록/삭제

### 5.2 클라이언트 상태 (Zustand)
| 스토어 | 용도 | 영속성 |
|--------|------|--------|
| `matchingStepStore` | AI 매칭 9단계 답변/진행상태 | localStorage (사용자별) |
| `adoptionVerificationStore` | 입양 신청 10단계 폼 데이터 | localStorage (사용자별) |

## 8. PWA 설정

### 8.1 Service Worker (`public/firebase-messaging-sw.js`)
- Firebase JS SDK 12 기반 (`firebase-app-compat`, `firebase-messaging-compat`)
- 백그라운드 메시지 수신 (`onBackgroundMessage`)
- 알림 클릭 시 `action_url`로 이동 또는 앱 포커스
- Firebase 웹 config(apiKey 등)는 공개 키 — 보안 이슈 아님

### 8.2 Firebase 초기화 (`src/lib/firebase.ts`)
- `NEXT_PUBLIC_FIREBASE_*` 환경변수로 설정 주입
- `getFirebaseMessaging()` — 브라우저 환경에서만 Messaging 인스턴스 반환
- `isSupported()` 체크로 미지원 환경 안전 처리

### 8.3 FCM 토큰 관리 (`src/hooks/mutation/usePushToken.tsx`)
- Capacitor 네이티브(ios/android)는 기존 FCM, 웹 브라우저는 Firebase JS SDK
- `detectPlatform()` 기반으로 네이티브/웹 완전 분리
- 네이티브 앱에서 웹 푸시 로직 절대 안 탐

## 9. SEO

### 9.1 전역 설정 (`src/app/layout.tsx`)
- `metadataBase: new URL("https://mpz.kr")` — canonical 등 상대경로 자동 해석
- 전역 OG (title, description, image), Twitter Card
- Google/Naver 사이트 인증
- JSON-LD 구조화 데이터:
  - `Organization` — 사이트 조직 정보
  - `WebSite` — 사이트 검색 액션
  - `SiteNavigationElement` — 주요 네비게이션 링크

### 9.2 동적 메타데이터
- 동물 상세 (`list/animal/[id]/page.tsx`): `generateMetadata`로 품종, 나이, 사진 등 동적 OG
- 센터 상세 (`list/center/[id]/page.tsx`): `generateMetadata`로 센터명, 주소 등 동적 OG
- 커뮤니티 상세 (`community/[id]/page.tsx`): `generateMetadata`로 게시글 제목, 내용 등 동적 OG
- 동물/센터 상세에 JSON-LD (`AnimalShelter` 등) 포함

### 9.3 공개 페이지 layout.tsx (정적 메타데이터)
- `"use client"` page.tsx에서는 `export const metadata` 사용 불가
- 같은 폴더에 서버 컴포넌트 `layout.tsx`를 만들어 metadata export
- 적용 경로: `list/animal/`, `list/center/`, `community/`, `matching/`, `event/centers/`

### 9.4 robots.ts
- dev 환경: 전체 크롤링 차단 (`Disallow: /`)
- prod 환경: 공개 페이지 허용, 비공개 페이지(`/centerpage/`, `/my/`, `/login/` 등) 차단
- 네이버 Yeti 봇 별도 규칙

### 9.5 sitemap.ts
- 정적 페이지 + 동물/센터 상세 동적 생성 (24h 캐시)
- dev 환경은 빈 배열 반환

## 13. 주의사항 및 알려진 이슈

1. **이미지 서빙**: Supabase URL 직접 사용 (API 프록시 삭제됨). `next.config.ts` remotePatterns에 `*.supabase.co`, `*.kakaocdn.net`, `openapi.animal.go.kr` 등록
2. **타입 변환**: 백엔드(snake_case) ↔ 프론트엔드(camelCase) 변환은 `types/animal.ts`의 transform 함수 사용
3. **flex gap 사용 금지**: 웹뷰 일정 버전에서 미지원. `margin`(ml, mt 등) 또는 `space-y`/`space-x` 또는 `grid gap` 사용
4. **Auth 상태 깜빡임 방지**: `isAuthenticated` 기반 조건부 렌더링 시 반드시 `isLoading` 상태 체크. 로딩 중에는 스켈레톤/placeholder 표시, 초기값(로그인 버튼 등) 노출 금지
5. **`"use client"` 페이지에서 metadata export 불가**: 정적 메타데이터가 필요하면 같은 폴더에 서버 컴포넌트 `layout.tsx`를 만들어 분리
6. **Tailwind 예약어 충돌 주의**: 커스텀 색상/변수명이 Tailwind 예약어(`lg`, `xl`, `sm` 등)와 겹치면 스타일 오염. font-size는 `text-[18px]` 같은 고정값 사용
7. **QueryClientProvider 순서 주의**: `useQueryClient`/`useQuery`는 `QueryClientProvider` 하위에서만 사용 가능. 상위 Provider에서 직접 사용 시 에러 → `CustomEvent` 기반 통신으로 우회
