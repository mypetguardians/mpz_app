## 3. API 구조

### 3.1 API 엔드포인트 총괄

모든 API는 Django Ninja 기반이며 `/v1/` prefix 하에 동작합니다.

| 경로 | 앱 | 설명 |
|------|-----|------|
| `GET /` | - | Health Check (`{"ping": "pong"}`) |
| `/v1/auth/*` | user | 회원가입, 로그인, 토큰 갱신, 프로필 |
| `/v1/admin/*` | user | 관리자 전용 API |
| `/v1/kakao/*` | user | 카카오 OAuth 로그인/콜백 |
| `/v1/animals/*` | animals | 동물 CRUD, 목록, 필터, 이미지 |
| `/v1/adoptions/*` | adoptions | 입양 프로세스 전체 |
| `/v1/centers/*` | centers | 센터 관리, 동의서, 계약서 |
| `/v1/preset-questions/*` | centers | 프리셋 질문 템플릿 |
| `/v1/cloudflare/*` | cloudflare | 이미지 업로드/삭제 (R2) |
| `/v1/favorites/*` | favorites | 즐겨찾기, 성격검사 |
| `/v1/notifications/*` | notifications | 알림 CRUD, 푸시토큰 |
| `/v1/posts/*` | posts | 게시글 CRUD, 태그, 좋아요 |
| `/v1/comments/*` | comments | 댓글/답글 CRUD |
| `/v1/ai/*` | ai | AI 추천 |
| `/v1/feedback/*` | feedback | 사용자 피드백 |
| `/v1/banners/*` | banners | 배너 관리 |
| `/v1/notices/*` | notices | 공지사항 |

### 3.2 인증 방식
```python
# api/security.py - JWTAuth
Authorization: Bearer <access_token>  # 헤더 방식
# 또는 쿠키의 accessToken/access 값
```
- 알고리즘: HS256
- Secret: `DJANGO_SECRET_KEY`
- access_token 만료: 1년 (365일)
- refresh_token 만료: 2년 (730일)

### 3.3 페이지네이션
```python
# api/pagination.py - CustomPageNumberPagination
Input:  page (1-based), page_size (기본 10)
Output: {
    "data": [...],
    "count": 100,        # 전체 개수
    "totalCnt": 100,
    "pageCnt": 10,       # 전체 페이지 수
    "curPage": 1,        # 현재 페이지
    "nextPage": 2,       # 다음 페이지 (없으면 null)
    "previousPage": null # 이전 페이지
}
```

### 3.4 API 문서
- URL: `/<engine>/` (staff 로그인 필요)
- 접근 제한: `@staff_member_required` 데코레이터

## 13. 주의사항

1. **Banner 모델**: `save()` 메서드에 R2 자격증명이 하드코딩되어 있음 → 환경변수로 이전 필요
2. **토큰 만료**: access_token 1년, refresh_token 2년으로 매우 길게 설정됨
3. **PersonalityTest**: favorites 앱에 포함되어 있음 (별도 앱이 아님)
4. **공공데이터 동기화**: 정기적으로 `sync_public_data` 실행 필요 (크론잡 또는 QStash)
5. **WebSocket**: Redis가 없으면 WebSocket 기능 동작 안 함
6. **마이그레이션**: 배포 시 `migrate --no-input` 자동 실행됨
