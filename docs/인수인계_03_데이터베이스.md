## 1. 데이터베이스 환경

### 1.1 프로덕션 DB
| 항목 | 설정 |
|------|------|
| DBMS | PostgreSQL |
| 호스팅 | Neon (Serverless PostgreSQL) |
| 리전 | ap-southeast-1 (싱가포르, AWS) |
| 접속 | Pooler 모드 (`-pooler` suffix) |
| SSL | `sslmode=require&channel_binding=require` |
| 연결 URL | `postgresql://neondb_owner:...@ep-holy-paper-a1i7mynh-pooler.ap-southeast-1.aws.neon.tech/neondb` |

### 1.2 테스트/로컬 DB
| 항목 | 설정 |
|------|------|
| DBMS | SQLite3 |
| 파일 | `backend/db.sqlite3` |
| 조건 | `sys.argv`에 "test" 포함 시 자동 전환 |

### 1.3 연결 설정
```python
DATABASES = {
    "default": dj_database_url.config(
        default=DATABASE_URL,
        conn_max_age=300,        # 5분 커넥션 풀링
        conn_health_checks=True  # 커넥션 헬스 체크
    )
}
```

## 4. 데이터 흐름 패턴

### 4.1 공공데이터 동기화
```
정부 공공데이터 API → sync_public_data 명령어
  → Animal (is_public_data=True, public_notice_number 설정)
  → AnimalImage (공공 이미지 URL)
  → Center (public_reg_no로 매핑)
```

### 4.2 입양 프로세스 상태 전이
```
신청 → 미팅 → 계약서작성 → 입양완료 → 모니터링
  │                                        │
  └─ 취소 ←────────────────────────────────┘(중단)
```

### 4.3 모니터링 체크 로직
```
입양완료 → monitoring_started 설정
  → total_checks = monitoring_period / monitoring_interval
  → 주기적으로 AdoptionMonitoringCheck 생성
  → 게시글(Post) 확인 → posts_found 업데이트
  → 미제출/지연 시 알림(Notification) 발송
```

## 6. 마이그레이션 관리

### 6.1 마이그레이션 명령어
```bash
# 마이그레이션 생성
python manage.py makemigrations
python manage.py makemigrations <앱이름>

# 마이그레이션 적용
python manage.py migrate
python manage.py migrate <앱이름>

# 마이그레이션 상태 확인
python manage.py showmigrations
```

### 6.2 배포 시 자동 실행
```bash
# paracord_runner.sh에서 자동 실행
python manage.py migrate --no-input
```

### 6.3 주의사항
- 프로덕션 DB는 Neon PostgreSQL이므로 마이그레이션은 PostgreSQL 호환으로 작성
- 테스트 시에는 SQLite3로 자동 전환되므로 DB 특화 기능(JSONB 등) 사용 시 주의
- UUID 필드는 PostgreSQL에서 네이티브 UUID 타입, SQLite에서는 CHAR(32)로 저장

## 7. 백업 및 복구

### 7.1 Neon 제공 기능
- **Point-in-Time Recovery (PITR)**: Neon 대시보드에서 특정 시점 복구 가능
- **Branch**: 데이터베이스 브랜치 기능으로 테스트 환경 생성 가능

### 7.2 수동 백업
```bash
# pg_dump를 이용한 백업
pg_dump $DATABASE_URL > backup_$(date +%Y%m%d).sql

# 복구
psql $DATABASE_URL < backup_YYYYMMDD.sql
```

## 8. 성능 고려사항
