## 2. Railway 배포 환경

### 2.1 서비스 구성

| 서비스 | 설명 | Dockerfile | 포트 |
|--------|------|-----------|------|
| Backend (Django) | API 서버 | `django_dockerfile` | $PORT (Railway 할당) |
| Frontend (Next.js) | 웹 앱 | `nextjs_dockerfile` | $PORT (Railway 할당) |
| Redis | WebSocket Channel Layer | Railway 내장 Redis Stack | 6379 |

### 2.2 Backend Dockerfile (django_dockerfile)

```dockerfile
FROM python:3.12-slim-bullseye

# 시스템 의존성
RUN apt-get update && apt-get install -y \
    libpq-dev libjpeg-dev libcairo2 gcc

# 가상환경 및 의존성 설치
COPY requirements.txt .
RUN pip install -r requirements.txt

# 앱 코드 복사
COPY . /app
WORKDIR /app

# 정적 파일 수집 (빌드 시)
RUN DJANGO_SECRET_KEY=dummy python manage.py collectstatic --noinput

# 비루트 사용자 실행
USER django-user

# 서버 시작
CMD gunicorn cfehome.asgi:application \
    -k uvicorn.workers.UvicornWorker \
    --workers 4 \
    --bind 0.0.0.0:$PORT
```

### 2.3 Frontend Dockerfile (nextjs_dockerfile)

```dockerfile
FROM node:20-alpine

WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

CMD ["npm", "start"]
```

### 2.4 Railway 배포 설정

**django_railway.toml**
```toml
[build]
builder = "DOCKERFILE"
dockerfilePath = "./django_dockerfile"
watchPatterns = ["backend/requirements.txt", "backend/**", "django_railway.toml", "django_dockerfile"]
```

**nextjs_railway.toml**
```toml
[build]
builder = "DOCKERFILE"
dockerfilePath = "./nextjs_dockerfile"
watchPatterns = ["frontend/**", "nextjs_railway.toml", "nextjs_dockerfile"]
```

### 2.5 배포 자동 감지
- `watchPatterns`에 정의된 파일 변경 시 자동 재배포
- Backend: `backend/**` 변경 시
- Frontend: `frontend/**` 변경 시

## 3. 로컬 개발 환경 (Docker Compose)

### 3.1 구성

```yaml
# docker-compose.yml
version: "3.3"
services:
  frontend:
    build: ./frontend
    ports: ["3000:3000"]
    volumes: ["./frontend:/app"]

  backend:
    build: ./backend
    env_file: ./backend/.env
    ports: ["8000:8000"]
    volumes: ["./backend:/app"]

  nginx:
    image: nginx:latest
    ports: ["80:80"]
    volumes:
      - ./nginx/default.conf:/etc/nginx/conf.d/default.conf
      - ./backend/local-cdn:/app/backend/local-cdn
    depends_on: [frontend, backend]
```

### 3.2 Nginx 리버스 프록시

```nginx
# nginx/default.conf
server {
    listen 80;

    # Django API
    location /api/ {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # Django Admin
    location /admin/ {
        proxy_pass http://backend:8000;
    }

    # Next.js 정적 자산
    location /_next/ {
        proxy_pass http://frontend:3000;
    }

    # 프론트엔드 (기본)
    location / {
        proxy_pass http://frontend:3000;
    }

    # 정적 파일 (로컬 CDN)
    location /static/ {
        alias /app/backend/local-cdn/;
    }
}
```

### 3.3 Makefile 명령어

```makefile
up:              # docker-compose up
build:           # docker-compose up --build
makemigrations:  # docker-compose run --rm backend python manage.py makemigrations
```

## 4. 데이터베이스 서버

### 4.1 Neon PostgreSQL

| 항목 | 값 |
|------|-----|
| 서비스 | Neon (Serverless PostgreSQL) |
| 리전 | ap-southeast-1 (싱가포르) |
| 데이터베이스명 | neondb |
| 접속 모드 | Connection Pooler (PgBouncer) |
| SSL | require + channel_binding |
| 연결 URL | `postgresql://neondb_owner:***@ep-holy-paper-a1i7mynh-pooler.ap-southeast-1.aws.neon.tech/neondb` |

**Neon 주요 기능:**
- Serverless: 사용하지 않을 때 자동 스케일 다운
- Branching: DB 브랜치 생성으로 테스트 환경 구성
- Point-in-Time Recovery: 특정 시점 복구
- Connection Pooler: PgBouncer 내장

### 4.2 Django 연결 설정
```python
DATABASES = {
    "default": dj_database_url.config(
        default=DATABASE_URL,
        conn_max_age=300,        # 커넥션 5분 유지
        conn_health_checks=True  # 헬스 체크 활성화
    )
}
```
