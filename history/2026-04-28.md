## 목록 UX 버그 수정

### 스크롤 위치 복원 (동물 + 센터)
- **문제 1**: AnimalTab의 `apiParams` useEffect가 마운트 시 `sessionStorage.removeItem` 호출 → 저장된 위치 삭제
  - **수정**: `apiParamsInitRef`로 초기 마운트 skip, 실제 필터 변경 시에만 clear
- **문제 2**: `sessionStorage.removeItem`을 scrollTo 직후 즉시 호출 → ListLayout 리렌더로 컴포넌트 재마운트 시 재시도 불가
  - **수정**: rAF 안에서 `scrollTop > 0` 확인 후에만 삭제 + scrollRestoredRef 완료 처리
- **문제 3**: ListLayout의 `controlHeight` 측정이 `useEffect`(페인트 후) → 스크롤 복원 시 marginBottom이 아직 0이라 레이아웃 어긋남
  - **수정**: `useLayoutEffect`로 변경하여 페인트 전에 측정 완료
- **문제 4**: CenterTab의 `measureElement`가 측정 전/후 높이 차이 유발 → scrollTop 복원 부정확
  - **수정**: 카드 높이 고정(h-[63px]) + measureElement 제거 + estimateSize 63
- **교훈**: 버추얼 스크롤에서 scrollTop 복원의 정확도는 아이템 높이 일관성에 의존. measureElement 사용 시 복원 부정확 트레이드오프 인지 필수

### 검색/필터 hide/show 깜빡임
- **문제**: 뒤로가기 시 검색 영역이 visible→hidden 전환이 보임
- **수정**: useScrollVisibility에 useLayoutEffect로 초기 상태 보정
  - sessionStorage에 저장된 스크롤 있으면 `visible=false`로 시작
  - `initialized` 플래그로 초기 보정 전 transition 비활성화 (rAF 후 활성화)
  - 프로그래밍적 scrollTo의 큰 delta(>300px)는 방향 감지에서 무시
- **교훈**: opacity만 단독 사용은 깜빡임으로 보일 수 있음. 다만 사용자가 원하는 애니메이션 방식을 먼저 확인 후 변경할 것 (max-height로 바꿨다가 원래 opacity로 복원)

### 센터 탭 전환 시 목록 안 보이는 버그
- **원인**: `useAnimalSearch` 훅의 sessionStorage 복원이 센터 탭에서도 `isSearching=true` 설정

### 센터 검색 실시간 반영
