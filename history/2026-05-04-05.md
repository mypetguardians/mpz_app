# 2026-05-04~05 작업 내역

## 마펫쯔 ↔ 강아지학교 통합 + EC2 인프라 전환

같은 법인이 운영하는 두 서비스(마펫쯔 mpz_app + 강아지학교 doggy-school)를 단일 인증·DB·인프라 체계로 통합. 강아지학교는 라이브 X 상태라 마펫쯔 기준 자유 재작성.

### 배경 / 결정

- 운영 주체: 마펫쯔 단일 법인. 개인정보 제3자 제공 동의 불필요, 처리방침/약관 통합.
- 유저 마스터: 마펫쯔 Django `public."user"` (UUID PK).
- Supabase 프로젝트: 이미 공유 중 (dev `djnjbimklqvzqgcrkrdf` / prod `uytovxdqmlmhdzzpmwzk`).
- 강아지학교 BE: Supabase 서버리스 유지 (Django로 합치지 않음).
- 인증: 마펫쯔 JWT 단일. Supabase Auth 폐기. JWT는 stateful → ACCESS 만료 15분으로 단축해 logout 갭 최소화.
- 강아지학교 전용 프로필: `user_school_profile` 1:1 테이블 (마펫쯔 user 테이블은 그대로).
- 클라이언트 범위: 강아지학교 = 웹 전용 (모바일 앱 WebView 미연동).
- 배포: Vercel 폐기 → 마펫쯔 EC2 docker-compose에 `school-frontend` 서비스로 통합.
- DNS: 가비아 (Route 53 아님). `memory/reference_dns_provider.md`에 영구 기록.

### Phase 3 — 강아지학교 코드 (PR #50)

- 75곳 `supabase.auth.getUser()` → `getCurrentUser()` 일괄 마이그레이션 (perl 멀티라인 정규식)
- `jose` 도입: `src/lib/auth/getCurrentUser.ts` (Edge Runtime 호환 JWT 검증, sub/exp/role 분기)
- Supabase 클라이언트 4개 재작성: 마펫쯔 `access` 쿠키를 `Authorization: Bearer`로 PostgREST에 forwarding. `admin.ts`(service_role)는 변경 없음 — HMAC 관리자 인증과 별개.
- 로그인/로그아웃: `signInWithOAuth`/`signOut` 폐기. 카카오 로그인은 `${MPZ_API}/v1/kakao/login?state=_frontend_<origin>_returnpath_<path>` 위임. 로그아웃은 `.mpz.kr` 쿠키 삭제 + 마펫쯔 `/v1/auth/logout` keep-alive POST.
- `auth/callback/` 디렉토리 삭제 (마펫쯔 BE가 콜백 처리)
- `src/middleware.ts`: 만료 임박 시 마펫쯔 `/v1/auth/refresh-token` 호출 + Set-Cookie forwarding
- 통합 마이그 단일 SQL 신설 + `auth.users` FK 12곳 → `public."user"` 변경. `comments` → `lesson_comments`, `categories` → `product_categories` 리네임 (마펫쯔 점유 테이블과 충돌 회피).
- RLS `is_admin()` 재작성: `EXISTS (SELECT 1 FROM public.user_school_profile WHERE user_id = auth.uid() AND role = 'admin')`

후속 정합화: PR #51 (login encoding 검증 위치 + Footer 단순화), #52 (사전 mismatch 4개 exclude), #53 (3개 정합화 + LAB_FEATURES 길이 4 복구), #54 (Route 53 → 가비아 정정).
